# SQL Customer Churn Project

A simple SQL project that analyzes customer churn using MySQL by studying customer behavior, transactions, support interactions, and engagement patterns.
The goal is to help businesses identify customers who are at risk of leaving and improve customer retention strategies.

# Problem Statement

Customer churn is a major problem for businesses because losing customers directly reduces revenue. Many companies struggle to:

Understand why customers stop using their services

Identify which customers are at high risk of churning

Track engagement levels, complaints, and spending patterns

Take timely action before customers leave

This project solves these challenges by:

Creating a structured SQL database for customer data

Analyzing transaction behavior, support issues, and activity

Segmenting customers based on usage and complaints

Assigning churn risk levels (Low, Medium, High)

Providing useful insights to support retention strategies


# How to Use

Install MySQL 8.0 or higher

Run churn_schema.sql to create all tables

Run churn_sample_data.sql to insert sample records

Run churn_analysis.sql to generate churn results

View insights and export results if needed

# Output & Analysis

After running the queries, you will get:

Customer activity level (Low / Medium / High)

Support complaint summary

Spending trends

Churn risk score for each customer

These insights help businesses take proactive actions to reduce churn.
